# AI 游资框架集合体数字人

这是一个本地可运行的前后端分离演示项目，放置在 `D:\shuziren`。

## 功能

- 用户输入股票相关问题，前端展示数字人、字幕、语音、动作。
- 后端用 FastAPI 处理问题理解、股票名称识别、行情查询、Prompt 调度和动作触发。
- 后端采用轻量 Agent 管线：任务识别 -> 标的解析 -> 指数环境 -> 个股行情 -> 结构评估 -> 预案生成。
- 行情、K 线、市场情绪和板块榜已加入 TTL 缓存，减少公开接口频繁请求和卡顿。
- 不接入大模型时，使用规则化决策评分引擎，输出明确的“回避/等待确认/纳入观察/可小仓试错/减压防守”等结论。
- 增加 RAG-lite 知识库、会话记忆、追问理解、用户情绪识别和多角色风格切换。
- 增加观察池：支持“加入观察池 / 移出观察池 / 查看观察池 / 复盘观察池”，并持久化触发条件和失效条件。
- 增加复盘模式：识别追高、补仓摊薄、仓位过重、退潮期进攻、非主线杂毛、无止损等常见错因。
- 增加情绪仪表盘：上涨/下跌、涨跌停、炸板率、连板高度、昨日涨停表现、样本覆盖率。
- 增加主线强度榜：综合板块涨幅、成交额、涨停数量和持续活跃天数评分。
- 增加回答置信度：展示行情、K 线、指数、情绪、板块、LLM 状态和是否盘中。
- 增加交易错误日志：复盘中的错因会记录到用户画像，后续类似问题会主动提醒。
- 增加数据口径标注层：每项数据标明来源类型、可用性、计算方法和覆盖率，区分公开行情、系统计算、模型生成。
- 增加风控一票否决层：满仓/梭哈、退潮进攻、被套补仓、极端杀跌抄底、空头趋势加仓等场景会覆盖原始结论。
- 增加 Agent 决策审计层：记录原始结论、最终结论、风控否决、证据链、置信度和数据口径到 `backend/decision_audit.jsonl`。
- 增加数据源健康监控：记录公开接口成功率、失败次数、最近错误、平均耗时，并在前端展示。
- 增加审计回放接口：可查看当前会话最近几次 Agent 判断，追溯当时证据。
- 增加观察池盘中提醒：定时刷新观察池，提示触发复核、失效降级或大幅波动。
- 增加 Agent 工具路由：按问题类型决定是否调用指数、情绪、个股行情、K线、板块工具，降低无关问题的延迟。
- 增加 Agent 工具并发执行：行情、K 线、情绪、板块按工具计划并发拉取，减少串行等待。
- 增加 Agent 离线评测集：`backend/agent_eval_cases.json` 固定覆盖意图、标的、工具路由、用户状态和风控场景。
- 增加流式进度接口：前端在完整回答返回前通过数字人动作展示“理解问题、工具路由、并发取数、风控校验、生成回答”等步骤。
- 后端拆出 `backend/agent_runtime.py` 和 `backend/llm_client.py`，并将聊天链路的 LLM 状态改为请求级结果，避免全局状态串扰。
- 后端继续拆出 `schemas.py`、`rules_config.py`、`llm_output.py`、`persistence.py`、`strategy_memory.py`、`agent_critic.py`、`tool_budget.py`，并提供 `agent.py`、`market_data.py`、`risk.py`、`memory.py`、`audit.py` 模块入口，降低主链路耦合。
- 核心意图、工具路由、风控阈值、仓位映射迁移到 `backend/rules/` 配置文件，便于不用改代码调整策略。
- LLM 输出增加结构化 JSON 校验：`verdict`、`reasoning_summary`、`trigger`、`invalid`、`position_limit`、`risk_flags`，不合格自动回退规则答案。
- 增加 Planner/Critic 自检：回答后检查确定性荐股、触发/失效条件、风控呈现和低置信度数据标注。
- 增加 SQLite 事件层 `backend/agent_state.sqlite3`，同步记录反馈和审计事件，为长期案例记忆/策略记忆做准备。
- 决策分档和工具预算继续配置化：`backend/rules/position_policy.json` 控制评分区间、机会仓位和动作口径，`backend/rules/tool_budget.json` 控制不同问题的工具优先级和降级提示。
- 增加用户画像摘要接口 `GET /api/profile/summary?session_id=xxx`，前端启动和每次回答后会刷新策略记忆、错因画像、观察池数量和反馈统计。
- 增强股票名称识别：对自然语言问题生成多个短词候选，减少“帮我看一下XX能不能买”这类句子无法识别股票的问题。
- 增加强制执行表：个股问题会输出空仓/持仓/触发/失效四类条件化动作，GLM 未输出时由规则后处理补齐。
- 增加“思维复刻”固定流程：市场情绪周期 -> 问题类型 -> 标的层级 -> 资金/情绪/筹码三维判断 -> 明确结论 -> 游资口径输出。
- 输入股票名称或 6 位 A 股代码时，尝试识别 A 股并通过腾讯/新浪公开接口获取实时或延迟行情。
- 可选接入 OpenAI 兼容大模型；不配置 API Key 时使用本地规则引擎兜底。
- 生成为多类游资公开短线框架融合后的结构化回复，包括行情、量价、风险、观察条件。
- 数字人通过表情、动作标签和浏览器语音合成进行表达。
- 内置合规提示：不冒充真实人物，不输出确定性荐股或收益承诺。

## 运行方式

### 方式一：一键启动（推荐）

双击运行：

```text
D:\shuziren\start.bat
```

脚本会自动：

- 创建 `.venv` 虚拟环境
- 安装后端依赖
- 启动 FastAPI 服务
- 提示访问 `http://127.0.0.1:8000`

服务启动后，可另开一个窗口运行自检：

```text
D:\shuziren\check.bat
```

### 方式二：手动启动

```bash
cd D:\shuziren
py -3 -m venv .venv
.venv\Scripts\activate
pip install -r backend\requirements.txt
uvicorn backend.main:app --reload --host 127.0.0.1 --port 8000
```

然后访问：

```text
http://127.0.0.1:8000
```

也可以直接双击打开 `index.html`，此时会自动使用纯前端降级模式。

如果只想开静态服务器，例如已安装 Node.js 时可以运行：

```bash
npx serve .
```

然后在浏览器访问命令行提示的本地地址。

## 文件说明

- `index.html`：页面结构和交互区域。
- `styles.css`：数字人形象、响应式布局、表情动画样式。
- `app.js`：前端交互、语音播报、数字人状态驱动、后端优先和本地降级逻辑。
- `backend/main.py`：FastAPI 后端，包含行情查询、Prompt、人设记忆、LLM 兼容调用和动作触发。
- `backend/requirements.txt`：后端依赖。
- `.env.example`：大模型、TTS、数字人 SaaS 的配置预留。

## 大模型接入

复制 `.env.example` 为 `.env`，填写 OpenAI 兼容接口配置后再启动后端。未配置时系统仍可运行，但使用本地规则回复。

未接入大模型时，系统是“工具型 Agent + 规则生成”；接入大模型后，系统会把 Agent 工具结果、行情、指数环境、人设记忆一起交给模型生成更灵活的回答。

当前已配置 GLM-4.5-Flash，使用智谱 BigModel OpenAI 兼容接口：

```text
OPENAI_BASE_URL=https://open.bigmodel.cn/api/paas/v4
OPENAI_MODEL=glm-4.5-flash
GLM_DISABLE_THINKING=1
```

密钥保存在本地 `.env`，并已通过 `.gitignore` 忽略，避免误提交。

已默认关闭 GLM thinking 模式，减少空回复和超时概率。如果 GLM 接口返回 `429` 或服务繁忙，后端会自动重试；仍失败时自动降级到本地 Agent 规则引擎。页面右侧“实时行情状态”会显示当前模型状态。

## 不接大模型时的决策逻辑

系统会根据以下因素打分：

- 指数环境：上证、深成指、创业板整体强弱。
- 个股强弱：涨跌幅、日内位置、振幅。
- 风险状态：是否高波动、是否靠近日内低位、是否极端下跌。
- 流动性：成交额是否足够。
- 用户状态：是否被套、是否已有较高仓位、是否给出成本价。

输出会包含：规则结论、评分、仓位上限、加分依据、扣分风险、触发条件、失效条件和散户处理建议。

## 智能增强模块

- `backend/knowledge_base.json`：轻量游资知识库，不用向量数据库也能按关键词检索。
- 会话记忆：记住上一次讨论的股票、成本、仓位，用户追问“那它还能拿吗”时会沿用上下文。
- 角色切换：情绪周期型、龙头进攻型、风控防守型、低位试错型。
- 情绪识别：焦虑时优先给风控动作，兴奋时主动降温，疑惑时拆步骤。
- 反馈闭环：每条回答支持“有用/没用”，记录到 `backend/feedback.jsonl`。
- 观察池闭环：观察标的、触发条件、失效条件和最新状态会写入 `backend/user_profiles.json`。
- 主线持续性：板块持续活跃天数会写入 `backend/sector_history.json`，该文件已忽略提交。
- 决策审计：每次普通聊天会写入 `backend/decision_audit.jsonl`，用于复盘“当时为什么这样判断”。

## Agent 审计和风控

系统现在把 Agent 决策拆成三层：

- 数据口径标注层：说明每项数据来自公开行情、系统计算、模型表达还是交易所官方；当前多数实时行情和短线情绪指标属于公开行情 + 系统计算，不伪装成官方统计。
- 风控一票否决层：在规则评分之后、LLM 润色之前执行。如果触发否决，最终结论会覆盖为“风控否决”，仓位上限强制为 `0 成`。
- 决策审计层：保留原始评分结论、最终结论、风控原因、证据链、数据口径、回答置信度，方便盘后回看。
- 工具路由层：先判断问题是否真的需要远程行情。仓位纪律、概念解释类问题会跳过行情工具；个股交易、市场情绪、观察池复盘才调用相关数据源。

触发风控否决的典型场景：

- 满仓、梭哈、重仓冲动。
- 退潮期还要求买入、加仓、追高。
- 被套后补仓摊薄。
- 个股极端杀跌时抄底。
- 空头趋势中继续加仓。
- 非主线标的高风险追击。

## 健康监控和提醒

新增接口：

- `GET /api/source-health`：查看数据源成功率、耗时、最近错误。
- `GET /api/audit/latest?session_id=xxx&limit=5`：回放最近审计记录。
- `GET /api/watchlist/alerts?session_id=xxx&role=cycle`：刷新观察池并返回触发/失效提醒。
- `GET /api/chat/progress?question=xxx&session_id=xxx&role=cycle`：SSE 流式返回 Agent 执行进度，驱动前端数字人动作。

前端会每 90 秒轮询观察池提醒和数据源健康。提醒只提示“复核/降级/大幅波动”，不等于交易指令。

## Agent 自动评测

每次修改 Agent 识别、风控、工具路由或用户状态解析后，建议运行：

```bash
.venv\Scripts\python.exe -m backend.run_agent_eval
.venv\Scripts\python.exe -m backend.run_response_quality_eval
.venv\Scripts\python.exe -m backend.test_agent_units
.venv\Scripts\python.exe -m backend.api_smoke_test
```

当前评测集包含 35 条离线用例，不依赖外部行情接口，主要检查：

- 意图识别是否稳定。
- 股票代码/本地别名是否识别正确。
- 工具路由是否按问题类型启用或跳过。
- 成本、半仓、满仓、被套等用户状态是否解析正确。
- 满仓/梭哈/补仓摊薄等风控是否触发。

## 观察池与复盘

可以直接用自然语言操作：

- `把豫能控股加入观察池`
- `查看观察池`
- `复盘观察池`
- `把 001896 移出观察池`
- `我这笔追高错在哪，帮我复盘`

观察池不是买入池。系统只保存条件化预案：触发条件先出现才重新评估，失效条件先出现则降级或移除。

观察池会在页面启动和查看时自动复盘：如果评分转弱、跌幅扩大或规则结论变成“明确回避/减压防守”，会提示“建议移除/降级”；如果进入“可小仓试错”，只提示触发复核，不直接给买入指令。

## 情绪仪表盘和主线强度

右侧面板会显示：

- 情绪阶段、上涨/下跌、涨停/跌停、炸板率、连板高度、昨日涨停表现。
- 样本数、覆盖率和数据来源，避免把部分样本误认为全市场。
- 主线强度榜，综合涨幅、成交额、涨停数、持续活跃天数，而不是只看板块涨幅。
- 回答置信度，说明当前回答的数据完整度和是否处于 A 股交易时段。

注意：炸板率、连板高度、昨日涨停表现依赖公开接口可用字段和 K 线估算，适合做情绪参考，不等同于交易所官方统计。

## 去机械感优化

- 思考延迟：根据问题长度动态等待，数字人先进入“抱臂思考”状态。
- 逐句播报：不再整段一次性朗读，按句子播放，中间有自然停顿。
- 打断能力：用户开始输入新问题时，会立刻停止当前语音和动作。
- 微动作循环：待机时自动眨眼、呼吸、轻微眼神偏移。
- 动作映射：风险/核按钮/吃面触发警示动作，机会/龙头触发指屏动作，必须/纪律触发强调动作。
- 黑话词典：识别核按钮、吃面、吃肉、龙头、杂毛、换手、卡位等短线黑话。
- 错误承认：用户指出判断错误时，系统会先承认可能看错，再按新数据重算。

## 思维复刻实现

系统不是只模仿口头禅，而是把游资集合体的思考顺序写进代码：

- 底层认知：市场情绪第一，技术第二，基本面最后；不预测，只应对；退潮期管住手。
- 固定 6 步：周期判断、问题类型、标的层级、资金/情绪/筹码、明确结论、游资口语输出。
- 层级判断：疑似核心/前排、板块前排观察、普通观察、弱势跟风、风险样本。
- 输出稳定：先给规则结论，再给依据、触发条件、失效条件和散户处理建议。

## 走势判断增强

系统现在会额外拉取东方财富历史 K 线，并计算：

- MA5 / MA10 / MA20 均线结构。
- 近 20 日支撑位和压力位。
- 量能相对最近 5 日均量的变化。
- 趋势标签：多头趋势、转强观察、震荡趋势、弱势修复、空头趋势。
- 走势推演：向上触发条件、向下失效条件、中性处理方式。
- 置信度：根据趋势评分给出 35%-85% 的区间。

注意：这不是保证预测正确，而是用更多交易证据提高判断质量。

## 五项优先级升级

- 板块和情绪数据：新增 A 股涨跌分布、涨跌停估算、板块涨幅榜、所属板块热度；情绪温度采用东方财富分页全量统计，并返回样本数/覆盖率。
- 决策面板：前端右侧展示最终评分、仓位上限和分项加减分。
- 用户画像：会话信息、关注股票、成本仓位、反馈次数会持久化到 `backend/user_profiles.json`。
- 大模型职责收窄：GLM 只负责把规则引擎结论润色成自然表达，不负责核心买卖判断。
- 数字人表现：保留思考延迟、逐句播报、打断、微动作、动作关键词映射，并新增可视化决策面板。

## 常见问题

- 如果提示 `python` 不可用，安装 Python 3.10+，或确认 Windows 的 `py -3` 可用。
- 如果依赖安装慢，可更换 pip 镜像，或等待 `start.bat` 自动安装完成。
- 如果浏览器打开后提示后端未启动，确认 `start.bat` 窗口没有关闭，并访问 `http://127.0.0.1:8000/health` 检查。
- 如果行情中文在命令行里乱码，不影响网页显示；网页使用 UTF-8，行情接口按 GBK 解码。

## 注意

本项目是研究型演示，是多游资公开短线框架的集合体，不是任何真实人物本人、官方账号或真人授权形象。系统不会复刻任何真实人物的私人决策，也不提供确定性荐股、买卖点承诺或收益承诺。所有回答仅用于学习交流。
